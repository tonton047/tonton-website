我花了一些时间学习如何使用纯CSS代码来绘制图形。事实上，这已经不是什么新鲜事物了。 你可以在CodePen或其他代码分享平台上搜索“draw”和“CSS”关键词，浏览更多大师们创作的精美图案，甚至是动态图形

这个瓶子看起来似乎非常简单，如果在专业的绘图软件上，可能只需要不到半小时就能完成。然而，通过CSS的方式，除了学习的时间，我花了大约两天的时间。 在绘图软件上，你的图形元素排列是可视化的，但在代码的世界中，它是这样呈现的

为什么这么做？

我找到了一个有趣的出发点，答案是：这个瓶子是响应式的

如果你现在正在手机上浏览这篇文章，可以尝试90°翻转手机，来看看瓶子是否发生变化 （如果你是在电脑上浏览这篇文章，可以通过自由拉伸浏览器窗口大小的方式来观察变化）

没错！这个瓶子能根据你当前屏幕的视窗比例，变得"更强壮"或变得"更纤细"

"响应式"是指网页或应用程序可以根据用户的设备和屏幕尺寸动态调整布局和显示效果， 以提供最佳的用户体验，这种设计方法旨在确保用户无论使用什么设备，都能够获得良好的可视化和操作体验。

自适应的界面设计已经渗透到我们生活的方方面面。它不仅仅是文字、按钮等元素放大缩小的调整， 更是基于媒体大小进行更合理的重新布局。当然，这不是一篇教学文章，因为我已经快要编不出其它内容了。

绘制这个瓶子运用到的几个知识点：

vw(宽)/vh(高)

"vw"和"vh"是非常巧妙的计量单位。另外还有其他一些智能单位，比如(dp、rem等)。使用方式如height:20vh，这表示元素的高度等于视窗高度的20%。 我可以通过一张图演示20px*20px和weight:20vw;height:20vh之间的区别


填充方式：linear-gradient、radial-gradient

linear-gradient

例:在一个大小为50vw*50vh的元素内，使用了线性渐变：linear-gradient(90deg, #BADBE7 0, #7BBADE 100%) 50% 0% / 50% 15%,可视化效果呈现为:

这个解读的文字含义是：使用90度方向的线性渐变，从颜色A（#BADBE7）开始，到颜色B（#7BBADE） 结束。居中显示，并且相对于画布宽度的50%，高度的15%进行定位


radial-gradient

例如:在一个大小为50vw*50vh的元素内，使用了径向渐变：`radial-gradient(100% 100% at 100% 0%, #FFFFFF 0, #7BBADE 100%)`。可视化效果呈现为：

这个解读的文字含义是：径向渐变，中心点位于画布的水平100%，垂直0%的位置。 渐变的颜色从白色（#FFFFFF）开始，到颜色B（#7BBADE）结束。颜色B占据整个渐变的100%位置


计算方式

calc( A +/- B )

calc( A +/- B )，我在这次绘制瓶子的实际操作中，在计算公式里使用了非常复杂的混合方式....(没想到它居然还奏效了), 例如height:calc(70vh - 11vh - 12% + 2px)，它的文字解读是高度=70%屏占比 - 11%屏占比 - 12%父级元素高度 + 2像素

注意 +/- 两侧添加空格
为什么不把'70vh - 11vh'直接写成59vh，这些数值一一对应需要计算的元素大小，方便进行对照修改


在前期的准备工作中，我任然需要通过绘图软件先进行构图，然后用css进行还原，这是一个不错的训练方式

Why did I do this?

I found an interesting starting point, and the answer is: this bottle is responsive.

If you're currently viewing this article on your phone, try rotating it 90° to see if the bottle changes (If you're on a computer, you can observe the changes by freely resizing the browser window).

That's right! This bottle can become "more robust" or "slimmer" based on the aspect ratio of your current screen viewport.

"Responsive" refers to the ability of a website or application to dynamically adjust its layout and display effects based on the user's device and screen size, aiming to provide the best user experience. This design approach ensures that users, regardless of the device they use, receive a visually appealing and user-friendly experience.

Adaptive interface design has permeated various aspects of our lives. It involves more than just resizing elements like text and buttons; it entails a thoughtful rearrangement of layouts based on media size. Of course, this isn't a tutorial article, as I'm running out of other content to write...

The knowledge points applied in drawing this bottle:

vw(width)/vh(height)

"vw" and "vh" are very clever measurement units. There are also other intelligent units, such as (dp, rem, etc.). Usage looks like height: 20vh, indicating that the element's height is equal to 20% of the viewport height. I can illustrate the difference between 20px*20px and weight: 20vw; height: 20vh with an image.


Filling methods: linear-gradient, radial-gradient.

linear-gradient

Example: In an element with a size of 50vw*50vh, a linear gradient is applied: linear-gradient(90deg, #BADBE7 0, #7BBADE 100%) 50% 0% / 50% 15%, and the visual effect is displayed as:

The interpretation of this text is: Apply a linear gradient in the direction of 90 degrees, starting from color A (#BADBE7) to color B (#7BBADE). It is centered and positioned relative to 50% of the canvas width and 15% of the height.


radial-gradient

The interpretation of this text is: Apply a radial gradient within an element sized at 50vw*50vh, using `radial-gradient(100% 100% at 100% 0%, #FFFFFF 0, #7BBADE 100%)`. The visualization appears as follows:

The interpretation of this text is: Radial gradient with the center point at the position of 100% horizontally and 0% vertically on the canvas. The gradient transitions from white (#FFFFFF) at the starting point to color B (#7BBADE) at the ending point, with color B occupying the entire 100% position of the gradient.


Calculation method

calc( A +/- B )

"calc( A +/- B )" is a calculation method I used in the actual process of drawing the bottle. I employed a complex combination in the formula, and surprisingly, it worked. For example, "height: calc(70vh - 11vh - 12% + 2px)" can be interpreted as the height being equal to 70% of the screen height minus 11% of the screen height minus 12% of the parent element's height plus 2 pixels.

Add spaces on both sides of +/- for attention.
Why not directly write '70vh - 11vh' as 59vh? These values correspond to the sizes of the elements that need to be calculated, making it easier for reference and modification.
